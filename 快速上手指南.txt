📌 这是什么东西？
─────────────────
一个基于 DeepSeek AI 的 QQ 聊天机器人，接入 QQ 群聊和私聊，
被 @ 就会回复，可以联网搜索、读取网页内容、使用本地知识库。

自带桌面管理程序，点叉可以最小化到系统托盘后台运行。

📥 第一步：下载安装所需软件
─────────────────────────────

你需要提前装好这些东西（都是免费的）：

  ① QQNT（QQ 9.9.x）
     → https://im.qq.com 下载安装
     ※ 注意：不要用 QQ 经典版，必须用 QQNT

  ② Python 3.10+
     → https://www.python.org/downloads 下载
     ※ 安装时记得勾选「Add Python to PATH」
     ※ 推荐 3.11 版本

  ③ NapCat（QQ 机器人框架，已包含在项目中）
     → 本项目已自带 NapCat 的 DLL 和配置文件
     → 你**不需要**额外下载 NapCat
     → 注意：NapCat 需要配合 QQNT 使用

  ④ DeepSeek API Key（AI 的 API 密钥）
     → https://platform.deepseek.com 注册账号
     → 在 API Keys 页面创建一个 Key
     ※ 免费的，注册就送额度


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

⚙️ 第二步：安装配置
─────────────────────

打开 PowerShell 或 CMD，依次执行以下命令：

  1. 克隆仓库
     git clone https://github.com/nico10086/-bot.git
     cd -bot

  2. 创建 Python 虚拟环境并安装依赖
     python -m venv env
     env\Scripts\pip install -r requirements.txt

  3. 配置 API Key（复制 .env.example 为 .env，填入你的密钥）
     或用桌面版在 UI 界面中直接填写，保存即可

     ※ NapCat 已包含在项目中，无需额外下载
     ※ 只需要确保电脑上安装了 QQNT 即可


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🚀 第三步：启动猫娘
─────────────────────

双击 start_app.bat 启动桌面管理程序

  ① 点击「▶ 一键启动」
  ② NapCat 自动启动 → 窗口显示二维码
  ③ 用手机 QQ 扫码登录
  ④ 猫娘上线！

看到日志出现 [WS] 已连接！等待 QQ 消息... 就说明成功了。

点窗口右上角 ✕ → 猫娘最小化到系统托盘继续运行。
想重新打开 → 双击托盘中的猫爪图标。

❓ 常见问题
─────────────────

Q：报错「文件损坏，请重新安装QQ」？
  → 确认你装的是 QQNT（9.9.x），不是 QQ 经典版
  → 检查杀毒软件是否拦截了 NapCat

Q：二维码不显示？
  → 「一键启动」后等待几秒，二维码会自动出现在界面中

Q：扫码后没反应？
  → 手机 QQ 上可能需要点「确认登录」
  → 检查端口 8080 是否被其他程序占用

Q：猫娘不回复群消息？
  → 检查是否在群里 @了猫娘
  → 检查 .env 里 BOT_NAME 是否配置正确

Q：如何切换模型 / 性格？
  → 在桌面版的配置区直接选择，保存后重启 Bot

Q：如何更改 API Key？
  → 在桌面版直接填写，或编辑 .env 文件

Q：猫娘回复太死板？
  → 在配置中把 temperature 调高到 0.7~0.8
  → 或者换成 deepseek-v4-pro 模型

� 管理指令（群里发送，不需要 @猫娘）
────────────────────────────────────────

每个群都有独立的管理系统。

【首次设置主人】
  发送  /setup master
  → 第一个发送此指令的人成为本群主人

【主人添加管理员】
  发送  /setup add QQ号
  → 主人可以用此指令添加管理员

【主人移除管理员】
  发送  /setup remove QQ号

【查看群配置】
  发送  /setup info
  → 查看主人、管理员列表、记忆数量

【添加核心记忆】
  发送  /memory 要记住的内容
  → 主人和管理员可以用
  → 猫娘每次回复都会参考核心记忆
  → 例如：/memory 猫娘最喜欢吃草莓

【查看核心记忆】
  发送  /memory list

【清除核心记忆】
  发送  /memory clear
  → 仅主人可用
